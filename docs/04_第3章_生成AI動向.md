# 第3章 現在の生成AI（ジェネレーティブAI）の動向

---

## 3-1. 生成AIが出来ることと主なサービス

### 生成AIの5つの生成カテゴリ

| カテゴリ | 概要 | 関連キーワード |
|---|---|---|
| **テキスト生成AI** | 文章・コード・翻訳等を生成 | ChatGPT, Claude, Gemini |
| **画像生成AI** | テキストや画像から新しい画像を生成 | リサイズ、正規化、データの水増し（augmentation）、データ拡張技術、リマスタリング |
| **音楽生成AI** | 楽曲・BGM等を自動生成 | - |
| **音声生成AI** | テキストから音声を合成 | - |
| **動画生成AI** | テキストや画像から動画を生成 | Sora, 自己回帰モデル, Veo3 |

### 画像処理の重要用語
- **画像のリサイズ**: 画像の解像度・サイズを変更
- **正規化**: データの値を一定の範囲（0〜1等）に変換する前処理
- **データの水増し（augmentation）**: 回転・反転等でデータ量を人工的に増やす手法
- **データ拡張技術**: augmentationの総称
- **リマスタリング**: 既存コンテンツの品質を向上させる処理

### 動画生成AIの注目サービス
- **Sora**: OpenAIの動画生成AI
- **Veo3**: Googleの動画生成AI（2026年シラバスで新規追加）
- **自己回帰モデル**: 動画生成でも使われる、前のフレームから次を予測する手法

## 3-2. ディープフェイク（深層偽造）技術

### ディープフェイクとは
- ディープラーニングを使って、実在の人物の顔や声を精巧に偽造する技術
- 正式名称: **ディープフェイク（深層偽造）技術**

### ディープフェイクの危険性
- **偽情報（ディスインフォメーション）**: 意図的に作られた虚偽情報の拡散
- 政治家の偽動画、詐欺、なりすまし等の犯罪に悪用されるリスク

### ディープフェイクによる事件
- 実際に発生した詐欺事件や選挙への影響等を理解しておく

## 3-3. RAG（検索拡張生成）★2026年新規追加

### RAGとは
- **RAG（Retrieval-Augmented Generation）**: 外部の知識ベースから関連情報を検索し、それをLLMの生成に組み合わせる手法
- LLM単体の弱点（知識の古さ、ハルシネーション）を補完する

### RAGの歴史と発展
- Meta AI（旧Facebook AI Research）が2020年に提唱
- LLMの普及とともに企業での活用が急速に拡大

### RAGの仕組みとメリット

```
ユーザーの質問
    ↓
① 検索（Retrieval）: 質問に関連する文書をベクトルデータベースから検索
    ↓
② 拡張（Augmented）: 検索結果をプロンプトに追加
    ↓
③ 生成（Generation）: LLMが検索結果を参考に回答を生成
```

### 重要用語
- **チャンク**: 文書を検索しやすい小さな単位に分割したもの
- **ベクトルデータベース**: テキストを数値ベクトルに変換して格納・検索するDB

### RAGのメリット
- 最新情報を反映できる（LLMの学習データに依存しない）
- ハルシネーションの低減
- 社内文書など独自データの活用が可能
- モデルの再学習が不要

### RAGのユースケース
- 社内ナレッジベースの検索・回答
- カスタマーサポートの自動化
- 法律・医療等の専門分野での情報検索

## 3-4. AIエージェント ★2026年新規追加

### AIエージェントとは
- 自律的に目標を設定し、計画を立て、ツールを使って行動するAIシステム
- 単なるチャットボットとの違い: 自律的な判断・行動・ツール利用が可能

### AIエージェントの仕組み
1. 目標の理解・分解
2. 計画の立案
3. 外部ツールの呼び出し・実行
4. 結果の評価・修正

### AIエージェントのツール事例

| ツール | 特徴 |
|---|---|
| **GenSpark** | AI検索エージェント |
| **Manus** | 汎用AIエージェント |
| **Skywork AI** | 中国発のAIエージェントプラットフォーム |

### MCPと外部連携
- **MCP（Model Context Protocol）**: AIモデルが外部ツールやデータソースと連携するための標準プロトコル
- AIエージェントが様々な外部サービスと安全に接続するための仕組み
