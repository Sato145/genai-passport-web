# 第5章 テキスト生成AIのプロンプト制作と実例

---

## 5-1. LMとLLM

### LM（Language Model: 言語モデル）
- 言語の確率的なパターンを学習し、次に来る単語を予測するモデル
- **n-gramモデル**: 直前のn個の単語から次の単語を予測する古典的手法
- **ニューラル言語モデル**: ニューラルネットワークを使った言語モデル

### LLM（Large Language Model: 大規模言語モデル）
- 大量のテキストデータで学習した巨大な言語モデル（GPT-4, Gemini, Claude等）
- **プレトレーニング**: 大量の汎用データで事前学習すること

### ハイパーパラメータ

| パラメータ | 説明 |
|---|---|
| **Temperature** | 出力のランダム性を制御。低い→確定的、高い→創造的 |
| **Top-p** | 累積確率がp以下の候補から次の単語を選択。多様性を制御 |

### プロンプトとプロンプトエンジニアリング
- **プロンプト**: AIに与える指示・入力テキスト
- **プロンプトエンジニアリング**: 望む出力を得るためにプロンプトを設計・最適化する技術

---

## 5-2. プロンプティングの基礎

### プロンプトの4要素

| 要素 | 英語 | 説明 |
|---|---|---|
| 指示 | **Instruction** | AIに何をしてほしいかの命令 |
| 文脈 | **Context** | 背景情報や前提条件 |
| 入力データ | **Input Data** | 処理対象のデータ |
| 出力指示 | **Output Indicator** | 出力の形式や条件 |

### 2つの基本プロンプティング手法

| 手法 | 説明 | 例 |
|---|---|---|
| **Zero-Shot プロンプティング** | 例示なしで指示のみ与える | 「以下の文を要約してください」 |
| **Few-Shot プロンプティング** | いくつかの例を示してから指示 | 「例: 入力→出力 のように変換してください」 |

## 5-3. LLMプロンプティングの実践

テキスト生成AIで実行できる基礎的なタスク:

### テキスト加工系
| タスク | ポイント |
|---|---|
| **文章の校正、校正箇所の確認** | 誤字脱字・文法ミスの修正。修正箇所の明示も可能 |
| **文章の整理** | 構成を整え、読みやすくする |
| **文章の要約** | 長文を短くまとめる |
| **箇条書き⇔文章の変換** | 形式の相互変換 |

### 表現変更系
| タスク | ポイント |
|---|---|
| **文章の対象を変更する** | 読者層に合わせた表現に変更（子供向け→専門家向け等） |
| **話者の設定を変更する** | 口調やキャラクターを変更 |
| **文章を会話のやり取りへ変換** | 説明文を対話形式に変換 |
| **例え話で理解を深める** | 抽象的な概念を身近な例で説明 |

### データ処理系
| タスク | ポイント |
|---|---|
| **数字の変換** | 単位変換、進数変換等 |

## 5-4. テキスト生成AIを用いたビジネス応用

### コミュニケーション
- **メールの作成**: ビジネスメールの下書き生成
- **海外企業宛のメール文章の作成**: 英語ビジネスメールの作成
- **外国語の翻訳**: 多言語翻訳
- **英単語から英文の作成**: キーワードから文章を構成

### 企画・分析
- **アンケート項目の作成**: 調査設計の支援
- **アンケートの分析**: 回答データの分析・要約
- **キャッチコピーの作成**: マーケティング用コピーの生成
- **ブレインストーミング**: アイデア出しの支援

### 業務効率化
- **ビジネス書類のテンプレート作成**: 定型文書の雛形生成
- **アジェンダの作成**: 会議の議題・進行表の作成
- **業務の手順を分解**: 複雑な業務をステップに分解
- **タスクの抽出**: プロジェクトからタスクを洗い出し

### その他
- **ディベートを行う**: 賛成・反対の論点を整理
- **姓と名の分離**: データクレンジング
- **ふりがなの記載**: 漢字にふりがなを付与
- **質問させながら一緒に進める**: 対話的にタスクを進行

## 5-5. テキスト生成AIの不得意なこと

| 不得意なこと | 理由 |
|---|---|
| **正確な文字数の指定** | トークン単位で処理するため、文字数の正確な制御が困難 |
| **計算** | 言語モデルは数学的計算が苦手（改善傾向にはある） |
| **最新の情報** | 学習データのカットオフ以降の情報を持たない |
| **芸術の批評** | 主観的・感性的な評価は苦手 |
